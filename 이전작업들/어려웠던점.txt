1. 팀프로젝트로 인한 버전 차이&import의 문제.
각자 맡은 개발을 진행하여도 버전차이와 import한 모듈에 대해 공유가 되지 않아서 테스트 과정에 어려움이 조금 있었습니다. 예를들어 gensim 안의 pyLDAvis는
현재 pyLDAvis.gensim은 -> pyLDAvis.gensim_model로 명명이 변경되었으나 이전 버전의 경우와 이름이 다르기 때문에 혼선이 발생했었습니다.
2. 1.에 이어서 모듈문제로 Konlpy같은 특정 모듈의 설치문제로 어려움을 겪었습니다.
한글을 분석하여 토픽을 뽑기위하여 사용하였던 Konlpy모듈이 설치문제로 상당히 많은 시간을 잡게되었습니다. 해당 Konlpy의 경우 java와 python을 연결시켜주는 부분에서 시스템 환경변수에 JAVA_HOME을 제외한 jdk를 가리키는 중복된 경로가 존재할경우 Anaconda3의 라이브러리 설정파일중 jvm.py파일에서 중복된 경로들이 등록되어 설치는 정상적으로 진행되었다고 생각하였으나, okt함수를 사용하지 못하는 먹통이 되어버리는 상태가 발생하였었습니다. 해당 문제를 검색하여도 잘 나오지 않아 이 또한 많은 시간이 소요되었습니다.
3. 공용api사용에 대한 내용입니다. api포맷에 맞춰서 코드를 작성하여도 api를 제공하는 서버에 문제가 발생하는게 빈번하다는 것을 알았습니다. 방금전까지 잘 작동하던 소스코드에서 변경도 하지 않았음에도 불구하고, 해당 api서버의 서버문제로 url요청이 무산되거나 첫url요청은 가능하나 데이터를 요청하는 과정에서 빈번한 오류가 발생하였었습니다.
4. 뉴스기사 본문을 크롤링할때의 문제입니다. 특정한 뉴스 페이지들은 자동화된 데이터 크롤링 문제를 차단하는것을 알게되었습니다. 이러한 페이지들은 사용자 agency를 구글로 설정하여 들어가면 되는것을 배웠습니다. 저희 소스에서는 다양한 페이지를 크롤링 하기 때문에 중첩된 뉴스들이 많다고 판단하여 해당 부분은 제거하였습니다.
5. 데이터를 크롤링할때 방대한 양의 데이터를 크롤링하고 분석하는것은 매우 좋은 일이나 모델을 통하여 학습을 시킬경우에 해당 시간이 상당히 오래 소요된다는것이 문제가 되었었습니다. 컴퓨터 사양의 문제로 저희는 데이터를 조금 축소하여 진행하였습니다.
